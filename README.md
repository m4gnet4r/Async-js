## 🧠 Concepts Explained

### 🔹 Synchronous vs Asynchronous JavaScript
Synchronous JavaScript executes code line by line, where each operation must complete before the next begins, which can lead to blocking behavior.  
Asynchronous JavaScript, on the other hand, allows certain operations (like API calls or timers) to run in the background, enabling non-blocking execution and better performance.

---

### 🔹 Callback Functions
Callbacks are functions passed as arguments to other functions and are executed after a task is completed.  
They were one of the earliest ways to handle asynchronous operations in JavaScript, but excessive nesting of callbacks can lead to complex and hard-to-read code, commonly known as *callback hell*.

---

### 🔹 Promises
Promises provide a more structured way to handle asynchronous operations.  
A promise represents a value that may be available now, later, or never. It has three states: **pending**, **fulfilled**, and **rejected**.  
Promises improve readability through chaining and simplify error handling using `.catch()`.

---

### 🔹 Async/Await
Async/Await is a modern syntax built on top of promises that allows asynchronous code to be written in a synchronous style.  
It improves code clarity and makes error handling easier using `try...catch`, making asynchronous logic more intuitive and maintainable.
